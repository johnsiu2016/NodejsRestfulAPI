    Introduction

    Welcome to the MeetUp API! This is the API docs for meetUp project, written by John Siu.

    Authentication

    API Key

    To test the authentication API key, use this code:

    # You can provide your key through header or get param or post body

For header,
curl "https://young-cove-95141.herokuapp.com/api/xxxxx" -H "apikey: API_KEY"

For get param,
curl "https://young-cove-95141.herokuapp.com/api/xxxxx?apikey=API_KEY"

For post body,
curl"https://young-cove-95141.herokuapp.com/api/xxxxx" -d "apikey=API_KEY"

    MeetUp uses API keys to allow access to the API.

    MeetUp expects for the API key to be included in all API requests to the server through the following three ways: (1) Header, or (2) GET param, or (3) POST body

    User Authorization

    To test the Authorization of user request, use this code:

    
curl "https://young-cove-95141.herokuapp.com/api/xxxxx?apikey=API_KEY" -H "Authorization: JWT USER_TOKEN"

    MeetUp uses JSON Web Token(JWT) to allow access to the user resource.

    MeetUp expects for the JWT to be included in all user-specific API requests to the server through the Authorization header

    This endpoint updates user account information.

    HTTP PATCH Request

    PATCH https://young-cove-95141.herokuapp.com/api/members/:member_id

    PATCH Parameters

    Parameter Optional Description
    email Yes Valid email
    name Yes Any characters, and cannot exceed 15 characters
    gender Yes Can only be one of the following value [‘male’, ‘female’, ‘other’]
    location Yes Any characters, and cannot exceed 150 characters
    phone Yes Numeric number, and is in the range of 8 to 11 characters.
    website Yes Valid URL, and cannot exceed 200 characters.
    avatar Yes MongoDB ID.

    duration No event duration (unit: hours), eg: 72
    fee No event fee (unit: HKD)
    status No event status, one of the [“cancelled”, “upcoming”, “past”, “proposed”, “suggested”, “draft”]

    Errors

    This error section is stored in a separate file in `includes/_errors.md`. Slate allows you to optionally separate out your docs into many files…just save them to the `includes` folder and add them to the top of your `index.md`‘s frontmatter. Files are included in the order listed.

    The Kittn API uses the following error codes:

    Error Code Meaning
    400 Bad Request – Your request sucks
    401 Unauthorized – Your API key is wrong
    403 Forbidden – The kitten requested is hidden for administrators only
    404 Not Found – The specified kitten could not be found
    405 Method Not Allowed – You tried to access a kitten with an invalid method
    406 Not Acceptable – You requested a format that isn’t json
    410 Gone – The kitten requested has been removed from our servers
    418 I’m a teapot
    429 Too Many Requests – You’re requesting too many kittens! Slow down!
    500 Internal Server Error – We had a problem with our server. Try again later.
    503 Service Unavailable – We’re temporarily offline for maintenance. Please try again later.
